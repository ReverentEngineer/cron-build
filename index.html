NAME

cron-build

SYNOPSIS

cron-build [options] [repo_file]

DESCRIPTION

cron-build runs script all the branches of a git repository and keeps track of their last status. The configuration is performed with two files.

The repo_list expected to be in the form:

<NAME> <TAB> <URL> <TAB> <SCRIPT> <NEWLINE>

The config_file is only necessary for mail notifications.

OPTIONS

--config=config_file

Configuration file

--jobs=count

Number of concurrent builds to run.

--mailto=recipients

Recipients to mail build status.

--verbose

Enables verbose output

CONFIGURATION FILE

Values
mailhost

The hostname of the SMTP server

mailfrom

The sender of mail notifications

mailuser

The name of the user account on the SMTP server

mailpass

The password to the user account on the SMPT server

EXAMPLES

Given a repo_file like this:

cron-build https://github.com/ReverentEngineer/cron-build make

cron-build would clone https://github.com/ReverentEngineer/cron-build and run make. The name is used for logging purposing and mail notifications.

Given a config_file like this:

mailhost=localhost mailfrom=root

cron-build will send notifications using the local SMTP server.
